After `shadcn init` (Task 1), `dealdrop/app/globals.css` contains a `.dark { ... }` class selector block that applies dark tokens only when an ancestor has `.dark` (typically from `next-themes` toggle). CONTEXT.md discretion chose "follow system, no toggle" — so wrap the `.dark` block contents in `@media (prefers-color-scheme: dark)` targeting `:root` directly.

**Step A — locate the `.dark { ... }` block in `app/globals.css`.**

Typical post-init shape (PATTERNS.md § "Pattern 5"):

```css
@import "tailwindcss";
@import "tw-animate-css";

@custom-variant dark (&:is(.dark *));

@theme inline {
  /* token → var mapping — KEEP as-is */
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  /* ... many more ... */
}

:root {
  --radius: 0.5rem;
  --background: oklch(1 0 0);
  --foreground: oklch(0.141 0.005 285.823);
  /* ... light tokens ... */
}

.dark {
  --background: oklch(0.141 0.005 285.823);
  --foreground: oklch(0.985 0 0);
  /* ... dark tokens ... */
}

@layer base {
  * { @apply border-border outline-ring/50; }
  body { @apply bg-background text-foreground; }
}
```

**Step B — replace the `.dark { ... }` block with a media query wrapper around `:root`:**

Change FROM:
```css
.dark {
  --background: oklch(0.141 0.005 285.823);
  --foreground: oklch(0.985 0 0);
  /* ...remaining dark tokens... */
}
```

TO:
```css
@media (prefers-color-scheme: dark) {
  :root {
    --background: oklch(0.141 0.005 285.823);
    --foreground: oklch(0.985 0 0);
    /* ...remaining dark tokens — copy verbatim from the .dark block... */
  }
}
```

Keep every token name/value identical — only the selector wrapper changes. The `@custom-variant dark (&:is(.dark *))` line above can stay (does no harm without a `.dark` ancestor) OR be removed. Leaving it is safer in case Phase 7 adds a manual toggle.

**Step C — verify font variables preserved.**

If the `shadcn init` wiped the Geist font variables that `layout.tsx` references (`--font-geist-sans`, `--font-geist-mono`), restore them. The layout sets them on `<html>` via `geistSans.variable` / `geistMono.variable` — they should still work because `next/font` injects them dynamically. But double-check the pre-init backup (`globals.css.before-shadcn`) for any additional tokens that were dropped.

If the pre-init file had a `@theme inline` block mapping `--font-sans: var(--font-geist-sans)`, merge that into the post-init `@theme inline` block. The base scaffold had:

```css
--font-sans: var(--font-geist-sans);
--font-mono: var(--font-geist-mono);
```

These should be inside the `@theme inline` block (Tailwind v4 picks them up automatically via `--font-*` prefix).

Do NOT:
- Delete the `@custom-variant dark` line — harmless, and a future toggle keeps working.
- Touch any tokens inside `:root { ... }` (light tokens) — leave them exactly as Shadcn wrote them.
- Remove the `@layer base { ... }` block — `body { @apply bg-background text-foreground }` is what makes Shadcn components look correct globally.
- Wrap `:root { ... }` (light tokens) in a media query — light is the default.

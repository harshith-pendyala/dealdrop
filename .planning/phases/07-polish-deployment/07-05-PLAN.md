<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator → Vercel CLI | Sensitive secrets (service_role, FIRECRAWL_API_KEY, RESEND_API_KEY, CRON_SECRET) are pasted into Vercel via the CLI; the CLI uses HTTPS to Vercel API. The four sensitive values use the `--sensitive` flag (write-once, non-readable from dashboard). |
| Operator → Supabase Dashboard | The prod CRON_SECRET is pasted into the Supabase SQL Editor (HTTPS). Vault encrypts at rest; only `vault.decrypted_secrets` (queryable inside SECURITY DEFINER scope) returns plaintext. |
| Vercel `production` env → Edge runtime | Server-only vars (SUPABASE_SERVICE_ROLE_KEY, etc.) are NOT prefixed `NEXT_PUBLIC_*` and never reach the client bundle (env.server.ts split from Phase 3 Plan 03-04). |

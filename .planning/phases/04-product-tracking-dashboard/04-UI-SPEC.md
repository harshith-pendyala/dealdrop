## Spacing Scale

Declared values (multiples of 4 only). Source: existing codebase patterns (`px-4`, `py-12`, `gap-6`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`h-6 w-6`), inline badge padding |
| sm | 8px | Compact form field stack spacing |
| md | 16px | Default element spacing; horizontal page padding (`px-4`) |
| lg | 24px | Card internal padding (`p-6`), grid column gap |
| xl | 32px | Section vertical rhythm |
| 2xl | 48px | Page top padding (`pt-12` = 48px) |
| 3xl | 64px | Major section breaks (hero `mt-16` = 64px) |

Exceptions:
- Touch targets for icon-only buttons (Trash2 remove): minimum 44px hit area via `p-2` + `h-9 w-9`
- Dashboard header height: 56px (`h-14`) — inherited from Header component

## Interaction Contracts

### Add Product Form Interactions

| State | Input appearance | Button appearance |
|-------|-----------------|-------------------|
| Idle | Border `--border`, placeholder text visible | `variant="default"`, enabled |
| Focused | Ring `ring-2 ring-ring` (shadcn default) | — |
| Validation error (client) | Border `ring-destructive` | disabled |
| Submitting | Normal, `disabled` attribute | disabled, spinner (`Loader2 animate-spin h-4 w-4`) |
| Success | Reset to idle (form clears) | Re-enabled |

URL input: `type="url"` is NOT used (too strict for pasted URLs). Use `type="text"` with client-side Zod validation on submit.

### Optimistic Skeleton Lifecycle

1. User submits valid URL → `SkeletonCard` appended to grid at position 0 (prepend, not append)
2. Server Action in flight → submit button shows spinner, input disabled
3. `{ok: true}` → `revalidatePath('/')` triggers re-render → skeleton replaced by real `ProductCard`
4. `{ok: false}` → skeleton removed from client state → `toast.error(message)` shown via Sonner

### Duplicate URL Ring Animation

On `{ok: false, reason: 'duplicate_url'}`:
1. `toast.error("You're already tracking this product.")`
2. Grid scrolls to the existing card (find by URL match in client state)
3. Card receives `ring-2 ring-primary` class
4. After 2000ms, ring class removed (plain `setTimeout` — no library needed)

### Remove Flow

1. Click `Trash2` icon button → opens `RemoveProductDialog`
2. User clicks "Remove" in AlertDialog → card removed from client state immediately (optimistic)
3. `removeProduct` Server Action called → on success `revalidatePath('/')` confirms
4. On Server Action error → card reinserted at original position, `toast.error("Couldn't remove that product. Try again.")` shown

### Show Chart Toggle

Phase 4 ships the toggle button only. The chart area below the card footer is:
- Hidden by default
- On click: renders an empty `div` with `min-h-[200px] bg-muted rounded-lg` as a placeholder
- Button label toggles: "Show Chart" ↔ "Hide Chart"
- Toggle state is local to the card component (`useState`) — not persisted

### Unauth Paste Flow

1. Logged-out user fills URL input and clicks "Track"
2. Client detects no session → writes URL to `sessionStorage['dealdrop:pending-add-url']`
3. Calls `openAuthModal()` from `AuthModalProvider` context
4. After OAuth callback, `DashboardShell` `useEffect` on mount reads key, clears it, auto-submits
5. If auto-submit fails → URL pre-populated in form input, `toast.error("Sign-in succeeded, but the add failed. Please try again.")` (graceful degradation)

## Accessibility Contracts

- Remove icon button (`Trash2`): must have `aria-label="Remove product"` — no visible text label
- "View Product" link: must have `rel="noopener noreferrer"` and `target="_blank"`; screen reader announcement uses visible text "View Product" (acceptable, no additional `aria-label` needed)
- "Show Chart" / "Hide Chart" toggle: `aria-expanded={isOpen}` on the button
- AlertDialog: Radix/shadcn primitives handle focus trap and `aria-modal` natively — no extra work
- Skeleton cards: `aria-hidden="true"` on the skeleton card wrapper so screen readers do not announce placeholder content
- Form submission state: `aria-disabled="true"` on input + button during pending state (mirrors `disabled` attribute for AT compatibility)

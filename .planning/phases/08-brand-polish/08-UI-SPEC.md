## Spacing Scale

Declared values — multiples of 4, standard 8-point scale. Phase 8 touches only
header, hero section, and dashboard cards. No spacing structure changes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge padding |
| sm | 8px | Button inner gaps (e.g. spinner + label), form input gap |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Hero subhead top margin, FeatureCard icon→title gap |
| xl | 32px | Header `mt-12` equivalent, grid column gap |
| 2xl | 48px | Hero top/bottom padding (pt-12 = 48px on sm) |
| 3xl | 64px | Page-level section breaks |

Exceptions:
- Header height: 56px (h-14 = 3.5rem). This is a locked inherited value from v1.0 — do NOT change.
- Logo height in header: 32px. Fixed per CONTEXT.md D-03.
- Touch targets: minimum 44px height on interactive elements (buttons already meet this via shadcn defaults).

## Typography

Existing type scale is preserved — Phase 8 does not change font sizes or weights.
The scale is catalogued here as a reference contract so the checker can confirm
no regressions are introduced during the brand restyle.

Exactly 4 semantic roles are declared. "Price" is an alias of Subheading — same
size and weight, accent color applied — and is NOT a fifth independent role.

Exactly 2 font weights are declared: 600 (font-semibold) for display/heading roles
and 400 (font-normal) for all text and label roles.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | `text-3xl md:text-5xl` (30px mobile / 48px desktop) | 600 (font-semibold) | 1.1–1.2 (leading-tight on mobile, leading-[1.1] on desktop) | Hero h1 headline — single responsive rule, not two separate roles |
| Subheading | 20px (text-xl) | 600 (font-semibold) | snug (~1.375) | FeatureCard title, EmptyState h1; **Price alias:** ProductCard current price in `text-primary` — same 20px/600 spec, orange accent applied |
| Body | 16px (text-base) | 400 (font-normal) | 1.625 (leading-relaxed) | Hero paragraph, FeatureCard blurb, EmptyState body |
| Label / Caption | 14px (text-sm) | 400 (font-normal) | default | Header wordmark, button labels, SignIn/Out buttons |

Note: orange-700 small-text accent (per D-05) is applied to FeatureCard blurb text
(`text-muted-foreground` override → `text-orange-700` in light mode) as the
primary surface for small-text brand warmth after the "Made with love" line is
deleted. FeatureCard card titles stay at `text-foreground`. The 12px Micro role
is deleted along with the "Made with love" line per BRAND-01.

## Color

### Visual Hierarchy

**Logged-out hero view:** The primary focal point is the Hero h1 headline
("Never miss a price drop") set in Display type on the orange-50 gradient
section. The eye travels from the headline → subline paragraph → "Track Price"
CTA button (orange-500 fill). Feature cards below are secondary reading.

**Logged-in dashboard view:** The primary focal point is the product grid. Within
each ProductCard, the orange-500 price figure is the dominant accent — it draws
the eye before the product title or action buttons. The header logo anchors top-
left; all other chrome is neutral zinc.

### Token Redefinition (BRAND-04 / D-06)

Phase 8 redefines `--primary` and `--primary-foreground` in `globals.css`.
All other Shadcn tokens (`--accent`, `--secondary`, `--muted`, etc.) are
**untouched**. The `--accent` token in Shadcn is a near-neutral hover background
— it is NOT the brand accent; do not confuse them.

#### Light mode `:root` values

| Token | Current Value | New Value | Notes |
|-------|--------------|-----------|-------|
| `--primary` | `oklch(0.21 0.006 285.885)` (zinc-900) | `oklch(0.646 0.222 41.116)` | Tailwind orange-500 in oklch |
| `--primary-foreground` | `oklch(0.985 0 0)` (zinc-50) | `oklch(0.985 0 0)` | Unchanged — zinc-50 passes AA on orange-500 |

#### Dark mode values (inside `@media (prefers-color-scheme: dark)`)

| Token | Current Value | New Value | Notes |
|-------|--------------|-----------|-------|
| `--primary` | `oklch(0.92 0.004 286.32)` (zinc-200) | `oklch(0.75 0.183 41.116)` | Tailwind orange-400 lifted lightness for dark bg legibility |
| `--primary-foreground` | `oklch(0.21 0.006 285.885)` (zinc-900) | `oklch(0.141 0.005 285.823)` | Near-black — passes AA on orange-400 |

Source: D-06, D-07 from 08-CONTEXT.md. Planner must verify exact oklch values
against Tailwind v4 default theme; these are representative targets.

### 60/30/10 Surface Split

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` = `oklch(1 0 0)` light / `oklch(0.141 0.005 285.823)` dark | All page surfaces, header background |
| Secondary (30%) | `--card` = same as background light / `oklch(0.21 0.006 285.885)` dark | ProductCards, FeatureCards, dialog panels |
| Accent (10%) | orange-500 via `--primary` (new) | See "Accent reserved for" below |
| Destructive | `--destructive` = `oklch(0.577 0.245 27.325)` light (unchanged) | "Remove" button / RemoveProductDialog only |

#### Accent reserved for (exhaustive list — do NOT expand without a new decision)

1. Primary button background: Sign In button, Track Price submit button (`bg-primary`)
2. Primary button hover state (`hover:bg-primary/90` → computed orange-600 equivalent)
3. ProductCard current price text (`text-primary` on the price `<p>`)
4. PriceChart line stroke (`stroke="var(--primary)"` — auto-cascades, no code change)
5. FeatureCard Lucide icon tint (`text-primary` on `<Icon>` — auto-cascades, no code change)
6. Hero section background: subtle orange-50 gradient on the `<section>` container (logged-out only)
7. Small-text accent: FeatureCard blurb text in `text-orange-700` light / `text-orange-400` dark

#### Hero gradient specification (D-09)

Apply to the `<section>` wrapping Hero content (logged-out view only):

```
bg-gradient-to-b from-orange-50 via-background to-background
```

Dark mode: orange-50 is very light and will appear near-invisible on dark backgrounds —
the planner should use `dark:from-transparent` or suppress the gradient class in dark mode
to avoid a near-black orange-tinted surface that harms legibility.

Hero heading and paragraph text must maintain WCAG AA contrast (4.5:1 minimum) against
the warmest gradient stop (orange-50). orange-50 is `#fff7ed` in hex — text-foreground
at zinc-900 passes easily.

| Token | Value | Usage |

**Phase-specific usage:**
- Chart slot outer padding: `px-4 pb-4` — matches the existing `{chartOpen && ...}` block in ProductCard (L59)

**Accent reserved for (Phase 5):**

**Chart color specifics:**
- Line stroke: `stroke="var(--primary)"` — `oklch(0.21 0.006 285.885)` light / `oklch(0.92 0.004 286.32)` dark
- Dot fill: `fill="var(--card)"` with `stroke="var(--primary)"` — matches card surface, visible against line
- Tooltip background: `var(--card)` with `var(--border)` outline
- Axis tick color: `var(--muted-foreground)` — `oklch(0.552 0.016 285.938)` light / `oklch(0.705 0.015 286.067)` dark

## Component Inventory

### New component: PriceChart

**File:** `dealdrop/src/components/dashboard/PriceChart.tsx`

```
Props: { history: PricePoint[]; currency: string }
Directive: 'use client'
```

**Focal point (expanded card state):** The `--primary` line stroke is the primary visual anchor within the expanded chart slot. All other chart elements (axis labels, tooltip chrome, dot outlines) are subordinate to it in visual weight.

**Recharts structure:**

```
<ResponsiveContainer width="100%" height={200}>
  <LineChart data={history} margin={{ top: 4, right: 8, bottom: 4, left: 0 }}>
    <XAxis
      dataKey="checked_at"
      tickFormatter={xTickFormatter}   // "MMM d" via Intl.DateTimeFormat
      tick={{ fontSize: 14, fill: 'var(--muted-foreground)' }}
    />
    <YAxis
      tickFormatter={yTickFormatter}   // Intl.NumberFormat, maximumFractionDigits: 0
      tick={{ fontSize: 14, fill: 'var(--muted-foreground)' }}
      domain={['auto', 'auto']}
      width={60}                       // leave room for formatted currency ticks
    />
    <Tooltip content={<PriceTooltip currency={currency} />} />
    <Line
      type="monotone"
      dataKey="price"
      stroke="var(--primary)"
      strokeWidth={2}
      dot={history.length === 1}       // show dot only for single-point case
      activeDot={{ r: 4, fill: 'var(--primary)' }}
    />
  </LineChart>
</ResponsiveContainer>
```

**Empty state (0 history rows — defensive only):**
```
<div className="min-h-[200px] bg-muted rounded-lg flex items-center justify-center">
  <p className="text-sm text-muted-foreground">No price history yet.</p>
</div>
```

**Single-point case:** Recharts renders a dot by default for a one-point Line. Set `dot={true}` explicitly when `history.length === 1`, otherwise `dot={false}` (let activeDot handle hover). No crash risk — verified by component tests.

## Interaction Contract

### Show/Hide Chart toggle (already shipped in Phase 4)

- Button label: "Show Chart" / "Hide Chart" with ChevronDown / ChevronUp icon — DO NOT re-wire; toggle logic is in ProductCard `chartOpen` state (L14-15)
- `aria-expanded={chartOpen}` is already applied (L46)
- Toggling `chartOpen` from `false` → `true` reveals `<PriceChart>` inside the `{chartOpen && ...}` block
- Chart data is pre-loaded (eager, server-side via DAL) — toggle is instant, no loading state needed

### Chart tooltip

- Triggered on hover / touch of any data point
- Shows: formatted date (`"April 20, 2026"` — full date in tooltip, short in axis ticks) + formatted price (full precision, e.g. `"£12.99"`)
- Tooltip uses `var(--card)` background, `var(--border)` border, `var(--foreground)` text
- On mobile touch, tooltip appears above the touch target; dismiss on tap-away

### Axis behavior

- X-axis: auto-tick selection by Recharts, short date format `"MMM d"` (e.g. `"Apr 20"`) — no year
- Y-axis: `domain={['auto', 'auto']}` (padded bounds, NOT zero-based) so small price drops are visible; ticks formatted without decimals to prevent crowding
- Both axes hide tick lines (`tickLine={false}`) and axis lines (`axisLine={false}`) for a minimal card appearance

### Responsive behavior

- `<ResponsiveContainer width="100%">` handles width — no breakpoint-specific chart code
- At 320px viewport (grid-cols-1), card is ~290px wide × 200px chart height — readable
- At desktop (grid-cols-3), card is ~300-350px wide × 200px — comfortable
- Y-axis width fixed at 60px to accommodate formatted currency labels on narrow viewports

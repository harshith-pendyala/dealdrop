Replace ONLY that three-line `select` block with this double-chain version (everything else in the file stays verbatim — do NOT touch `auth`, `insert`, `delete`, the interface, or the overrides handling):

```typescript
      select: vi.fn((_cols?: string) => ({
        // Supports both single .order() (Phase 4 flat select) and
        // double .order() (Phase 5 nested select with referencedTable).
        // The inner .order() resolves to selectProducts regardless of chain depth.
        order: vi.fn().mockReturnValue({
          then: (onFulfilled: (v: typeof selectProducts) => unknown) =>
            Promise.resolve(selectProducts).then(onFulfilled),
          order: vi.fn().mockResolvedValue(selectProducts),
        }),
      })),
```

**Why the `then` thenable:** Phase 4's existing `getUserProducts` DAL awaits `.select('*').order(...)` (one `.order`). Without a `then` on the first `.order()` return value, awaiting it produces `undefined` instead of `selectProducts` and breaks every currently-green Phase 4 test that uses `selectProducts` override. The `then` thenable makes the mock's first `.order()` result both awaitable (Phase 4 path) and chainable (Phase 5 path).

## Typography

**Fonts:** Geist Sans loaded as `--font-geist-sans` CSS variable; body inherits via `font-sans` (set in Phase 1 `@theme inline`). Geist Mono is loaded but unused in Phase 2.

**Exactly 4 sizes. Exactly 2 weights. No italics, no tracking overrides.**

| Role | Size | Weight | Line Height | Tailwind | Used for |
|------|------|--------|-------------|----------|----------|
| Display | 48px | 600 (semibold) | 1.1 | `text-5xl font-semibold leading-[1.1] tracking-tight` | Hero tagline "Never miss a price drop" (desktop) |
| Display (mobile) | 32px | 600 | 1.15 | `text-3xl sm:text-5xl font-semibold leading-tight` | Hero tagline clamps down <640px |

**Weight rule:** Only `font-normal` (400) and `font-semibold` (600) are allowed. `font-medium` (500) is permitted only inside Shadcn Button primitives (already baked into the variant) ŌĆö do not introduce `font-medium` on any other element. This keeps the weight contract at 2 + 1 primitive-owned.

## Interaction Contract

### OAuth Flow (happy path)

```
User clicks header "Sign in"
  ŌåÆ SignInButton calls openAuthModal()
  ŌåÆ AuthModalProvider sets isOpen=true
  ŌåÆ Dialog animates in (Shadcn default animation, ~200ms)
  ŌåÆ Focus lands on "Continue with Google" button
User clicks "Continue with Google"
  ŌåÆ Button shows loading spinner + stays disabled
  ŌåÆ supabase.auth.signInWithOAuth({ provider: 'google' }) fires
  ŌåÆ Browser redirects to Google
  ŌåÆ Google redirects to Supabase
  ŌåÆ Supabase redirects to /auth/callback
  ŌåÆ Route Handler exchanges code ŌåÆ writes session cookies ŌåÆ redirect to /
  ŌåÆ app/page.tsx RSC re-renders with session ŌåÆ shows DashboardShell
  ŌåÆ Modal is gone (component unmounted in the re-render)
  ŌåÆ No toast on success (dashboard appearing is the feedback)
```

### OAuth Flow (failure path ŌĆö user cancels or Google returns error)

```
User lands back at / with no session
  ŌåÆ app/page.tsx still shows Hero
  ŌåÆ Route Handler detected error and redirected to `/?auth_error=1`
  ŌåÆ Client reads query param on mount, calls toast.error("Sign in failed. Please try again."), then cleans the URL via router.replace('/')
  ŌåÆ Modal stays closed (user can retry from header)
```

Exact query-param contract: `?auth_error=1` ŌĆö no error code leakage to user, no specific reason surfaced. Portfolio bar.

### Sign-Out Flow

```
User clicks header "Sign out"
  ŌåÆ Button disables + label ŌåÆ "Signing outŌĆ”"
  ŌåÆ Server Action calls supabase.auth.signOut()
  ŌåÆ Server Action redirects to /
  ŌåÆ app/page.tsx RSC renders Hero
  ŌåÆ Client detects the post-signout state (query param `?signed_out=1` appended by the action) ŌåÆ toast.success("Signed out") ŌåÆ router.replace('/')
```

## Accessibility Floor (portfolio bar)

These are the only a11y guarantees Phase 2 makes. Exhaustive audit is deferred to Phase 7 Polish.

- All interactive elements are native `<button>` / `<a>` (no `<div onClick>`)
- Dialog trap + ESC + restore focus: inherited from Radix via Shadcn (free ŌĆö do not reimplement)

- Color contrast: zinc new-york preset ships WCAG AA ratios out of the box; do not introduce custom colors that could break this

- Tagline is a single `<h1>` per page ŌĆö both Hero and DashboardShell use `<h1>` (they're mutually exclusive)
- Lucide icons inside cards have `aria-hidden="true"` ŌĆö the card title is the accessible name, icon is decorative
